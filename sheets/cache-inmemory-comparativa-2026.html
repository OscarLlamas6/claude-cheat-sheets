01 Redis 8.x / 2025 — BSD + RSAL
TCP / RESP3 El estándar C
~1M ops/s
  • Estructura de datos in-memory: strings, hashes, lists, sets, sorted sets, streams, bitmaps, HyperLogLog
  • Persistencia: RDB snapshots + AOF (append-only file)
  • Pub/Sub, Lua scripting, transactions (MULTI/EXEC)
  • Redis Cluster: sharding automático horizontal
  • Redis Sentinel: HA con failover automático
  • Modules: RedisSearch, RedisJSON, RedisTimeSeries, RedisBloom
  • No es base de datos relacional ni soporta JOINs
  • Operaciones multi-key en cluster con limitaciones
  • Sin soporte nativo de tipos geoespaciales complejos
  • Licencia RSAL desde 7.4 (no OSS puro)
  • El caché distribuido más adoptado del planeta
  • Latencia sub-milisegundo consistente
  • Ecosistema de clientes en todos los lenguajes
  • Versatilidad: caché, cola, pub/sub, leaderboards, rate limiting
  • Redis 8: vector search nativo para AI workloads
  • Licencia RSAL desde v7.4 (cambio controvertido)
  • Single-threaded por comando (I/O multihilo, pero ejecución mono)
  • Memoria es el límite: datasets mayores que RAM = problema
  • Cluster: operaciones cross-slot son costosas
  • Caché de sesión, rate limiting, leaderboards
  • Pub/Sub para notificaciones en tiempo real
  • Colas de trabajo (RPUSH/BLPOP o Redis Streams)
  • Vector store para RAG / embeddings (Redis 8)
  • Caché distribuido en microservicios
  • Datos que excedan significativamente la RAM disponible
  • Queries analíticas complejas o joins
  • Cuando la licencia RSAL es un bloqueador legal
 RSAL (no OSS puro) Redis Cloud / Enterprise

  • Sin TTL por entrada individual
  • Sin colisión handling: colisiones sobrescriben
  • Sin persistencia
  • Pausa GC mínima: 1.5ms con 20M entradas vs 9ms de map nativo
  • Ideal para cachés de gigabytes sin degradación de GC
  • Diseño simple y predecible
  • Creado en producción real en Allegro (e-commerce escala)
  • Desarrollo lento: última release 2022
  • Sin TTL por entrada
  • Colisiones de hash sobrescriben datos silenciosamente
