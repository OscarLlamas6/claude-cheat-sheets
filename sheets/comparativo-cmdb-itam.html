¿Es esto
una CMDB?

Una comparación honesta entre NetBox, PacketFence y OCS Inventory — más tres alternativas que sí pertenecen al mismo barrio: GLPI, iTop y Snipe-IT.

La pregunta corta primero.

Una CMDB (Configuration Management Database) es un repositorio donde se modelan los Configuration Items (CIs) y, sobre todo, las relaciones entre ellos para soportar procesos ITIL (incidentes, cambios, problemas, impacto). Con esa definición en mente:

NetBox
CMDB de RED

Es una "Source of Truth" para infraestructura de red (DCIM + IPAM). Funciona como CMDB especializada, no ITIL genérica.

PacketFence
NO ES CMDB

Es un NAC (Network Access Control). Controla quién entra a la red. No es repositorio de configuración.

OCS Inventory
INVENTARIO/ITAM

Es un sistema de descubrimiento e inventario de activos. Alimenta una CMDB, pero no lo es.

// Conclusión: ninguna de las tres es una CMDB ITIL pura. Si eso es lo que buscás, mirá iTop o GLPI en las alternativas.

§ 02 — FICHAS INDIVIDUALES

Cada herramienta, en detalle

NetBox

DCIM + IPAM Open Source · Apache 2.0 · Python/Django

Qué es

La "Source of Truth" de facto para infraestructura de red. Funciona como CMDB especializada: el modelo de datos describe cómo es tu red, no qué pasa con cada laptop.

Qué no es

No es una CMDB ITIL genérica, no es helpdesk, no es sistema de monitoreo, no descubre dispositivos por sí mismo (es declarativo).

Qué hace

Modela racks, dispositivos, interfaces, cables, VLANs, prefijos IP, VRFs, ASNs, circuitos WAN, tenants, contactos. Su API REST y GraphQL son la razón principal por la que se volvió estándar: alimenta automatización con Ansible, Nornir, Terraform.

Qué no hace

No descubre topología automáticamente, no inventaría software de PCs, no gestiona tickets, no controla acceso a red, no monitorea estado.

Ventajas

  • API excepcional, ideal para Infrastructure as Code en redes
  • Modelo de datos riguroso y bien pensado
  • Comunidad enorme, ecosistema de plugins amplio
  • Estándar de facto en NetOps; la mayoría de herramientas modernas se integran

Desventajas

  • No descubre nada — los datos los ponés vos (manualmente o vía scripts)
  • Requiere disciplina operativa para mantener actualizado
  • Curva de aprendizaje real para aprovechar el modelo
  • No sirve para activos no-de-red (laptops, software, mobiliario)

Cuándo SÍ

ISPs, datacenters, equipos NetOps, automatización de red, MSPs con múltiples clientes de red, ambientes con cientos+ de dispositivos donde el Excel ya no escala.

Cuándo NO

Si tu objetivo principal es inventariar PCs, abrir tickets, controlar quién se conecta a la red, o si tu red cabe en una hoja de papel.

PacketFence

NAC Open Source · GPL v2 · Inverse Inc.

Qué es

Una solución de Network Access Control que decide quién puede o no conectarse a tu red, en qué VLAN cae y bajo qué condiciones.

Qué no es

No es una CMDB. No es inventario. No es DCIM. No es helpdesk. Es un guardia de seguridad en la puerta de la red.

Qué hace

Autenticación 802.1X, MAB (MAC Auth Bypass), captive portal, perfilado de dispositivos por DHCP fingerprinting, integración con AD/LDAP/RADIUS, asignación dinámica de VLAN, cuarentena de dispositivos no compliant, BYOD onboarding.

Qué no hace

No inventaría software de los equipos, no gestiona racks ni cables, no abre tickets ITIL, no monitorea aplicaciones.

Ventajas

  • NAC enterprise sin costo de licencia (vs Cisco ISE, Aruba ClearPass)
  • Soporta múltiples vendors de switches y APs
  • Capacidad real de impedir conexiones no autorizadas
  • Bueno para cumplimiento (PCI-DSS, HIPAA)

Desventajas

  • Implementación compleja: hay que tocar config de switches, RADIUS, CA
  • Requiere expertise sólido en redes y AAA
  • Mantenimiento operativo no trivial
  • Muchas piezas móviles (RADIUS, DB, captive, agentes)

Cuándo SÍ

Universidades, hospitales, ambientes con BYOD, redes que necesitan cumplimiento normativo, organizaciones con muchos invitados, segmentación por rol/dispositivo.

Cuándo NO

Si tu red es chica, si no tenés switches gestionables/802.1X-capable, si lo que necesitás es inventario o CMDB, o si no tenés a alguien que sepa de RADIUS.

OCS Inventory NG

ITAM / Discovery Open Source · GPL v2 · Perl/PHP

Qué es

Un sistema de descubrimiento e inventario de activos IT: un servidor central + agentes en los equipos que reportan hardware, software y configuración.

Qué no es

No es CMDB ITIL (no modela relaciones complejas entre CIs), no es ticketing, no es DCIM ni NAC.

Qué hace

Descubre y reporta: hardware (CPU, RAM, disco), software instalado, parches, antivirus, periféricos. Tiene IpDiscover (escaneo de red sin agente), despliegue de paquetes, y agentes para Windows, Linux, macOS, Android.

Qué no hace

No modela red (sin IPAM/DCIM), no tiene flujos ITIL ni tickets, no controla acceso, no genera mesa de ayuda.

Ventajas

  • Descubrimiento automático maduro y confiable
  • Agentes ligeros y multiplataforma
  • Integración nativa con GLPI (combo clásico ITSM)
  • Detecta software no autorizado y vence licencias

Desventajas

  • UI con look anticuado
  • Solo no llega a ser CMDB — necesita complementarse
  • Reporting limitado sin acompañarlo de GLPI
  • Mantenimiento de agentes en flotas grandes puede ser un dolor

Cuándo SÍ

Empresas que quieren saber qué software/hardware tienen, control de licencias, auditoría de cumplimiento, alimentar GLPI o iTop con datos reales.

Cuándo NO

Si necesitás CMDB ITIL completa, gestión de red, NAC o helpdesk. Tampoco si querés algo solo para tracking de activos físicos sin software (mejor Snipe-IT).

GLPI

ITSM + CMDB ALTERNATIVA Open Source · GPL v3 · Teclib

Qué es

Suite ITSM completa con helpdesk, gestión de activos, CMDB básica, contratos, licencias y proyectos. La opción "todo-en-uno" gratis del mundo open source.

Qué hace de más

Tickets con SLA, base de conocimiento, formularios, reservaciones de equipos. Combina con OCS o GLPI Agent (FusionInventory) para descubrimiento automático.

Ventajas

  • Suite completa: tickets + CMDB + activos en uno
  • Comunidad enorme y plugins para todo
  • Inventario nativo con GLPI Agent (ya no necesitás OCS aparte)
  • Multi-entidad para MSPs

Desventajas

  • UI mejor que antes pero aún recargada
  • CMDB no llega al rigor ITIL de iTop
  • Performance puede sufrir con cientos de miles de CIs

Cuándo elegirlo

PYMEs y medianas que quieren helpdesk + activos + CMDB ligera sin pagar licencias. Reemplazo natural del combo OCS+algo.

Cuándo no

Si necesitás CMDB ITIL pura, NAC, o modelado de red profundo.

iTop

CMDB ITIL ALTERNATIVA Open Source · AGPL v3 · Combodo

Qué es

La CMDB ITIL "de libro" en open source. Diseñada desde cero para modelar Configuration Items, sus relaciones y los procesos ITIL que los gobiernan.

Qué hace de más

Análisis de impacto sobre la topología de CIs (¿si cae este server, qué servicios mueren?), procesos ITIL completos (incidentes, problemas, cambios, release), modelado profundo de relaciones.

Ventajas

  • La CMDB open source más cercana a estándar ITIL
  • Modelado de relaciones de CIs sobresaliente
  • Análisis de impacto visual
  • Datamodel extensible

Desventajas

  • Curva de aprendizaje pronunciada
  • Implementación pesada — proyecto, no instalación
  • No descubre solo (necesita conector con OCS, ServiceNow, etc.)

Cuándo elegirlo

Empresas medianas/grandes con procesos ITIL formales que necesitan análisis de impacto, gestión de cambios estricta y CMDB de verdad.

Cuándo no

Si sos PYME, si no tenés procesos ITIL maduros, o si solo querés un inventario rápido.

Snipe-IT

Asset Management ALTERNATIVA Open Source · AGPL v3 · Laravel/PHP

Qué es

Gestor de activos físicos enfocado en tracking puro y duro: ¿dónde está cada laptop, monitor, licencia? ¿a quién está asignado? ¿cuándo se compró? ¿cuánto se depreció?

Qué hace de más

Flujos de checkout/checkin a usuarios, etiquetas QR/código de barras, recordatorios de auditoría, gestión de consumibles y accesorios. UI moderna y rápida.

Ventajas

  • UI moderna y agradable de usar
  • API limpia y bien documentada
  • Setup en horas, no en semanas
  • Foco preciso: no se dispersa intentando ser ITSM

Desventajas

  • Sin relaciones complejas estilo CMDB
  • Sin tickets ni helpdesk
  • Sin descubrimiento automático (es manual o vía API)
  • Sin gestión de red ni NAC

Cuándo elegirlo

Equipos IT pequeños/medianos que solo necesitan saber qué activos tienen y a quién están asignados. RR.HH. tech, MSP que entrega equipos a empleados.

Cuándo no

Si necesitás procesos ITIL, descubrimiento automático, tickets, modelado de red o CMDB con relaciones.

Decisión rápida

¿Necesito una "fuente de verdad" para mi red?
→ NetBox
¿Quiero controlar quién se conecta a mi red?
→ PacketFence
¿Necesito saber qué hay instalado en cada PC?
→ OCS Inventory (o GLPI Agent)
¿Quiero helpdesk + activos + CMDB básica todo en uno?
→ GLPI
¿Necesito una CMDB ITIL "de verdad"?
→ iTop
¿Solo quiero saber dónde está cada laptop?
→ Snipe-IT
¿Combo "definitivo" para una mediana empresa?
→ NetBox + GLPI + OCS
¿Combo "ITIL serio" para empresa grande?
→ NetBox + iTop + OCS + PacketFence
