01 OpenLDAP 2.6.x / slapd — 2025
LDAPv3 Linux / BSD Referencia OSS
  • Servidor LDAP de referencia en el mundo OSS
  • Motor MDB (LMDB) de alto rendimiento como backend por defecto
  • Soporte completo LDAPv3: replicación syncrepl, SASL, TLS
  • Overlays: memberOf, ppolicy, accesslog, syncprov, auditlog
  • Esquemas extensibles (OID personalizados)
  • Sin GUI de gestión oficial
  • Sin soporte Kerberos/KDC nativo (delegar a MIT Kerberos)
  • Sin consola web, SCIM o provisioning nativo
  • Sin soporte OAuth2/OIDC built-in
  • El servidor LDAP más extendido en Unix/Linux
  • LMDB: backend extremadamente rápido y confiable
  • Máxima flexibilidad de esquemas y ACLs
  • Sin licencia, sin costos, sin vendor
  • Enorme ecosistema de documentación y herramientas
  • Configuración vía LDIF (cn=config) muy arcana
  • Sin soporte comercial oficial
  • Curva de aprendizaje pronunciada
  • Documentación fragmentada y a veces desactualizada
  • Backend LDAP para aplicaciones Linux/Unix
  • Directorio de usuarios para sistemas internos
  • Base de autenticación para RADIUS, PAM, SSH
  • Laboratorios y entornos de aprendizaje
  • Organizaciones sin equipo técnico LDAP
  • Cuando se necesita UI de gestión amigable
  • Reemplazo directo de Active Directory (usar Samba o FreeIPA)
 OSS (OpenLDAP Public License) ✗
02 389 Directory Server 3.1.x — Red Hat / 2025
LDAPv3 RHEL / Fedora Base de FreeIPA
  • Servidor de directorio empresarial derivado de Netscape Directory Server
  • Replicación multi-master (Multi-Supplier), sync agreements
  • Web Console moderno (Cockpit plugin)
  • DNA plugin para autoincremento de UIDs/GIDs
  • Password policy, lockout, history built-in
  • Sin KDC Kerberos nativo (requiere MIT Kerberos externo o FreeIPA)
  • Sin SCIM nativo
  • Sin soporte OAuth2/OIDC built-in
  • Replicación multi-master robusta y probada en producción
  • Web Console usable sin línea de comandos
  • Base de datos Berkeley DB → LMDB (migración en curso)
  • Muy activo: Red Hat mantiene desarrollo continuo
  • Integración natural con FreeIPA y RHEL
  • Principalmente orientado a ecosistema Red Hat
  • Menos conocido fuera del mundo RHEL/Fedora
  • Configuración inicial más compleja que OpenLDAP básico
  • Directorio enterprise en entornos RHEL/CentOS/Fedora
  • Backend de FreeIPA en producción
  • Organizaciones que necesitan replicación multi-master estable
  • Entornos Debian/Ubuntu (OpenLDAP más natural)
  • Cuando se necesita Kerberos integrado (usar FreeIPA completo)
 OSS (GPL) Via Red Hat / RHDS

  • Trusts bidireccionales con Active Directory (AD Trust)
  • Web UI completa y CLI (ipa command)
  • No gestiona dispositivos Windows nativamente
  • Sin gestión de Microsoft 365 / Azure AD
  • No es IdP SAML/OIDC por sí solo (requiere Keycloak/Dex)
  • El "Active Directory" de Linux: solución todo-en-uno
  • Kerberos SSO transparente en toda la infraestructura Linux
  • AD Trust permite coexistir con entornos Windows
  • Replicación multi-master con 389DS de base
  • Gestión de certificados con Dogtag CA integrado
  • Instalación consume muchos recursos (no apto para VMs pequeñas)
  • RHEL/Fedora/CentOS recomendado; Debian/Ubuntu con fricción
  • Complejidad operacional alta en entornos heterogéneos
  • Infraestructuras Linux-centric con SSO Kerberos
  • Organizaciones con servidores RHEL y necesidad de PKI propia
  • Entornos mixtos Linux+Windows con AD Trust
  • Datacenter on-prem con gestión centralizada de identidades
  • Entornos 100% Windows (usar AD directamente)
  • Equipos pequeños sin experiencia en Kerberos
  • Cloud-native sin servidores on-prem
