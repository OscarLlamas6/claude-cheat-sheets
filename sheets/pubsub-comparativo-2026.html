RabbitMQ
v4.x · 2026 Broadcom / VMware
OSS FREE MANAGED SELF-HOST

Broker de mensajería tradicional basado en AMQP. Es el estándar de facto para colas de tareas y mensajería empresarial desde 2007. Implementa el modelo push hacia consumidores.

Ruteo flexible de mensajes con exchanges (direct, topic, fanout, headers). Acknowledgments, dead-letter queues, priority queues, TTL, RPC patterns, pub/sub clásico, quorum queues para HA.

No es un log distribuido; no hace replay histórico nativo. No diseñado para millones de msg/s sostenidos. No stream processing nativo (aunque Streams feature es limitada).
  • Ecosistema maduro y muy documentado
  • Flexibilidad de ruteo insuperable
  • UI de administración incluida
  • Soporte multi-protocolo (AMQP, MQTT, STOMP)
  • Quorum queues para durabilidad robusta
  • Throughput limitado vs Kafka (~50k-100k msg/s)
  • Broadcom adquirió y cambió licenciamiento enterprise
  • Escalado horizontal complejo
  • Sin replay de mensajes consumidos

Apache Kafka
v3.9 · 2026 Apache / Confluent
OSS FREE FREEMIUM SELF-HOST

Log distribuido de eventos. No es un broker tradicional sino un commit log particionado y replicado. El estándar del mercado para event streaming a gran escala.

Streaming de eventos con retención durable (días/semanas). Replay ilimitado. Particionado para escala horizontal. Kafka Streams y ksqlDB para stream processing. Kafka Connect para integración.

No es bueno para mensajes pequeños y muchas colas dinámicas. Latencia p99 alta. Sin ruteo sofisticado por mensaje. Gestión operacional compleja. ZooKeeper deprecado (KRaft en v3.x).

Strimzi
v0.45 · 2026 CNCF / Red Hat
OSS FREE SELF-HOST

Operador Kubernetes para Apache Kafka. No es un broker nuevo: es la capa de gestión/orquestación que convierte Kafka en un ciudadano nativo de K8s vía CRDs (Custom Resource Definitions).

No es independiente de Kafka: SIN Kafka no funciona. No añade capacidades de mensajería nuevas. No compite con RabbitMQ. No ofrece managed service propio.

Redis Pub/Sub
Redis 8.x · 2026 Redis Ltd / Valkey fork
OSS FREE FREEMIUM SELF-HOST

Sistema pub/sub fire-and-forget integrado en Redis. Incluye también Redis Streams (estructura de datos para event log). No es un MQ dedicado: es una feature de una base de datos en memoria.

Pub/Sub básico (PUBLISH/SUBSCRIBE/PSUBSCRIBE). Redis Streams: log append-only con consumer groups, ACK, replay limitado. Latencia ultra-baja (<1ms). Integrado con caching.

Pub/Sub nativo: sin persistencia ni ACK (fire-and-forget). Streams: sin escala horizontal nativa de particiones. Sin schema registry. Sin exactly-once semántica robusta. No reemplaza Kafka para durabilidad.

Plataforma de eventos + workflows durable. No es un MQ tradicional: combina event-driven messaging con orquestación de workflows (como Temporal + event bus). Diseñado para developers modernos.

Durable functions (pasos que sobreviven fallos), event-driven triggers, scheduled jobs (cron), fan-out, throttling, step functions, replay de eventos, local development server, observabilidad nativa.

No es un message broker de alto throughput. Sin Kafka API. No reemplaza Kafka/RabbitMQ para streaming masivo. Más caro que soluciones de infraestructura pura a escala.
