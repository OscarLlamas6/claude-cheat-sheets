CSPM

Cloud Security Posture Management

Monitorea y evalúa configuraciones de recursos cloud para detectar malas configuraciones, incumplimientos regulatorios y riesgos de exposición. Actúa sobre el plano de control, no sobre el runtime. Pregunta clave: ¿está bien configurado mi entorno?

AWS Security Hub + Config + Trusted Advisor

AWS
Servicios nativos AWS para CSPM
Security Hub CSPMAWS ConfigTrusted Advisor
¿Qué hace?
Security Hub agrega hallazgos y evalúa contra CIS, NIST, PCI-DSS, AWS FSBP. Config registra historial de cambios de configuración y evalúa reglas de compliance. Trusted Advisor detecta desviaciones de best practices en seguridad, costo y disponibilidad.
¿Qué NO hace?
No cubre Azure/GCP. Sin remediación automática nativa completa. Config escala en costo con muchos recursos. Sin Security Graph ni correlación de rutas de ataque. Trusted Advisor básico es limitado.
✅ Ventajas
Integración nativa AWS perfecta. Panel unificado. FSBP out-of-the-box. Config da historial de cambios. Trusted Advisor incluido en Basic sin costo.
❌ Desventajas
Solo AWS. Costo escala con findings y Config rules. Sin correlación de rutas de ataque. Hallazgos requieren integración (EventBridge+Lambda) para accionar.
Úsalo cuando
Tu infra es mayormente AWS y necesitas compliance centralizado con mínimo esfuerzo operativo.
No lo uses cuando
Tienes entorno multi-cloud o necesitas remediación automática avanzada y correlación de rutas de ataque.
PARTIAL FREEPAID por finding/config rule

Microsoft Defender for Cloud

Azure
Servicios nativos Azure para CSPM
Defender for CloudAzure PolicySecure Score
¿Qué hace?
CSPM nativo de Azure con Secure Score visual y accionable. Evalúa contra CIS, NIST, ISO 27001, PCI-DSS. Plan básico gratuito. Defender CSPM agrega Attack Path Analysis, CIEM, y cobertura multi-cloud (AWS/GCP via conectores).
¿Qué NO hace?

✅ Ventajas
Secure Score claro. Free tier útil. Soporte multi-cloud. Bien integrado con Sentinel. Attack Path Analysis en Defender CSPM. El más completo de los 3 grandes nativos.
❌ Desventajas
Planes avanzados costosos. Mejor en ecosistema Microsoft. Curva de aprendizaje alta en orgs grandes. Conectores multi-cloud tienen límites.
Úsalo cuando
Eres Azure-first o entorno híbrido Microsoft. Free tier es buen punto de partida.
No lo uses cuando
Tu entorno es 100% AWS/GCP sin Azure o el costo del plan avanzado no se justifica.

Wiz

Multi-cloud (CNAPP líder)
¿Qué hace?
CNAPP líder con Security Graph único. Correlaciona malas configuraciones + CVEs + identidades + rutas de ataque + datos sensibles. Sin agente. AWS, Azure, GCP, OCI. Mejor CSPM del mercado en correlación contextual. Lider Gartner CNAPP 2024.
¿Qué NO hace?
Sin versión gratuita ni POC sin contrato. No cubre on-premise. Precio muy alto (>$100K/año). CWPP/runtime menos profundo que Prisma Cloud.
✅ Ventajas
Security Graph único en mercado. Sin agente. Correlación de riesgos excepcional. Implementación en horas. Multi-cloud completo.
❌ Desventajas
Precio >$100K/año. Sin POC gratuito. Sin on-premise. CWPP runtime básico comparado con Prisma.
Úsalo cuando
Empresa mediana/grande, multi-cloud, que prioriza posture y correlación de riesgos sobre runtime/EDR.
No lo uses cuando
Presupuesto limitado, on-premise significativo, o eres startup pequeña.
PAID (Enterprise, >$100K/año)
CWPP

Cloud Workload Protection Platform

Protege cargas de trabajo (VMs, contenedores, serverless) en runtime. Detecta amenazas activas, malware y comportamientos anómalos mientras corren. Complementa al CSPM: si CSPM dice "¿está bien configurado?", CWPP dice "¿está siendo atacado ahora?"

Amazon GuardDuty

AWS
Servicios nativos AWS para CWPP
GuardDutyRuntime Monitoring (EKS/ECS/Lambda)Detective
¿Qué hace?
Detección de amenazas con ML y threat intelligence. Analiza CloudTrail, VPC Flow Logs, DNS. Detecta crypto-mining, exfiltración, accesos anómalos. GuardDuty Runtime para EKS/ECS/Lambda detecta comportamiento en contenedores en tiempo real.
¿Qué NO hace?
Solo detecta, no previene activamente. Sin remediación automática nativa. No cubre Azure/GCP. Sin EDR/AV tradicional. Runtime Monitoring requiere agente adicional en EKS.
✅ Ventajas
Sin agente en la mayoría de casos. ML avanzado. Bajo mantenimiento. Integración con Security Hub y EventBridge para respuesta automatizada.
❌ Desventajas
Solo AWS. Solo detección. Costo escala con volumen de logs. Runtime Monitoring adicional en precio.
Úsalo cuando
Eres AWS-native y quieres CWPP de detección con mínimo esfuerzo operativo.
No lo uses cuando
Necesitas prevención activa, multi-cloud o EDR real en workloads.

Falco

OSS / K8s / On-Premise
¿Qué hace?
Runtime security open source para Linux/contenedores/K8s via eBPF. Detecta comportamientos anómalos en syscalls: accesos a archivos sensibles, conexiones inesperadas, escalada de privilegios en tiempo real. CNCF graduated project.
¿Qué NO hace?
Solo detección, no prevención activa. Sin UI propia. Sin gestión centralizada SaaS out-of-the-box. Muchos falsos positivos si mal configurado. Requiere expertise para operar bien.
✅ Ventajas
Gratuito. CNCF graduated. eBPF moderno. Reglas personalizables. Funciona on-premise y en cualquier nube. Sin vendor lock-in.
❌ Desventajas
Requiere expertise K8s. Falsos positivos sin afinación. Sin UI incluida. Solo detección (combinar con Tracee para blocking).
Úsalo cuando
Tienes K8s on-premise o en nube y quieres runtime security gratuito y potente con control total.
No lo uses cuando
No tienes expertise en K8s o necesitas solución out-of-the-box con soporte comercial.

CIEM

Cloud Infrastructure Entitlement Management

Gestión de identidades y permisos en la nube. Detecta permisos excesivos (over-provisioning), genera políticas de privilegio mínimo y monitorea uso real vs. permisos otorgados. Pregunta clave: ¿quién puede hacer qué, y realmente lo usa?

IAM Access Analyzer + Credential Report + CloudTrail

AWS
Servicios nativos AWS para CIEM
IAM Access AnalyzerIAM Credential ReportAWS CloudTrailIAM Last Access Info
¿Qué hace?
IAM Access Analyzer detecta acceso no intencionado desde fuera de la cuenta y genera políticas least-privilege basadas en actividad real (últimos 90 días). Credential Report lista todas las credenciales y su última actividad. CloudTrail audita toda acción en la cuenta.
¿Qué NO hace?
Sin análisis de blast radius ni rutas de escalada complejas. Solo AWS. Análisis limitado a 90 días de actividad. Sin visualización de grafo de identidades. CIEM básico comparado con herramientas especializadas.
✅ Ventajas
Gratuito (Access Analyzer + Credential Report). Recomendaciones basadas en uso real. CloudTrail básico gratuito. Fácil activación. Detecta recursos expuestos a internet vía políticas.
❌ Desventajas
Solo AWS. Sin análisis de escalada de privilegios complejo. CloudTrail avanzado (Lake) es de pago. 90 días de historial son insuficientes para algunas auditorías.
Úsalo cuando
Eres AWS-only y quieres CIEM básico gratuito para detectar permisos excesivos y recursos expuestos.
No lo uses cuando
Necesitas análisis avanzado de rutas de escalada de privilegios o entornos multi-cloud.
FREE (Access Analyzer + Credential Report)PAID (CloudTrail Lake)

DSPM

Data Security Posture Management

Descubre, clasifica y protege datos sensibles en la nube (PII, PHI, PCI, secretos, IP). Evalúa quién tiene acceso a qué datos y si están protegidos. Foco en el dato. Pregunta clave: ¿dónde están mis datos sensibles y quién puede verlos?

Amazon Macie + KMS (+ Cyera via SecHub)

AWS
Servicios nativos AWS para DSPM
Amazon MacieAWS KMSS3 Object LockSecHub → Cyera (partner)
¿Qué hace?
Macie descubre y clasifica datos sensibles en S3 usando ML (PII, credenciales, datos de salud). Evalúa configuración de buckets (público, cifrado). KMS gestiona claves. Para DSPM completo, AWS recomienda integrar Cyera vía Security Hub Extended.
¿Qué NO hace?
Macie solo S3. No cubre RDS, DynamoDB, Redshift, SaaS nativamente. Sin análisis de flujo de datos. DSPM muy limitado sin integración de terceros (Cyera, BigID).
✅ Ventajas
Macie: ML muy preciso para PII en S3. Integración con Security Hub. KMS: gestión de claves robusta. Cyera: DSPM más completo del mercado.
❌ Desventajas
Macie solo S3. Precio escala con datos. DSPM nativo muy limitado. Cyera costoso. Sin análisis de acceso efectivo a datos.
Úsalo cuando
Datos sensibles en S3 y necesitas clasificación básica para compliance. Para DSPM completo, integra Cyera.
No lo uses cuando
Datos en RDS/DynamoDB/SaaS o necesitas DSPM multi-datastore completo.
FREE 30 días (Macie)PAID

Cloud Vulnerability Management

Descubre, prioriza y gestiona CVEs en workloads cloud. Va más allá del escaneo clásico correlacionando vulnerabilidades con contexto cloud (exposición a internet, criticidad del activo) para priorización inteligente. Incluye gestión de parches.

¿Qué NO hace?

CNAPP

Cloud Native Application Protection Platform

La convergencia de CSPM + CWPP + CIEM + KSPM + DSPM + CVM en una sola plataforma. Elimina el tool sprawl y correlaciona hallazgos cross-dominio para priorización más inteligente. Como dijo el profesor: CNAPP = CSPM + CWPP + los agregados que varían por vendor.

💡
Cada vendor tiene diferente fortaleza: AWS Security Hub v2 = agregador nativo; Defender for Cloud = mejor en ecosistema Microsoft; SCC Enterprise = mejor en GCP con Chronicle; Cloud Guard OCI = básico e incluido. Los vendors 3rd party (Wiz, Prisma, CrowdStrike) son los CNAPPs "puros" más completos del mercado.
