▸ IDENTIFICACIÓN DE VULNERABILIDADES
CVE Common Vulnerabilities and Exposures
Identificador Lista pública de vulnerabilidades y exposiciones de seguridad conocidas, cada una con un identificador único. Nombrar y rastrear vulnerabilidades de forma estándar entre diferentes plataformas, herramientas y equipos. MITRE Corporation
Patrocinado por el gobierno federal de EE. UU. 		CVE-ID → NVD enriquece CVE-ID → CVSS puntúa CVE-ID → CWE clasifica el tipo CVE-2024-1234
Año + número secuencial
CNA CVE Numbering Authority
Programa Organizaciones autorizadas por MITRE para asignar IDs de CVE a vulnerabilidades dentro de su ámbito (ej. Microsoft, Google, Red Hat). Descentralizar y agilizar la asignación de CVE-IDs sin que todo pase por MITRE. MITRE (autoriza a CNAs)
> 400 CNAs a nivel global 		Los CNAs crean entradas CVE que luego son indexadas por el NVD y puntuadas con CVSS.

▸ PUNTUACIÓN Y SEVERIDAD
CVSS Common Vulnerability Scoring System
Sistema de puntuación Framework estándar para evaluar la severidad de una vulnerabilidad mediante una puntuación numérica de 0 a 10. Priorizar qué vulnerabilidades requieren atención inmediata comparando su severidad de forma objetiva. FIRST
Forum of Incident Response and Security Teams 		CVSS puntúa cada CVE. El NVD publica la puntuación CVSS base de cada CVE. No asigna IDs.
None (0.0)
Low (0.1–3.9)
Medium (4.0–6.9)
High (7.0–8.9)
Critical (9.0–10.0)
CVSS
Métricas Base · Temporal · Ambiental
Sub-componente Las tres capas de cálculo del score CVSS: Base (cualidades intrínsecas), Temporal (disponibilidad de parches) y Ambiental (contexto del entorno). Afinar la puntuación de severidad según el momento y el entorno específico de cada organización. FIRST (define el estándar)
NVD reporta solo Base 		El NVD publica solo la métrica Base. Las métricas Temporal y Ambiental las calcula cada organización según su contexto.
Base: AV/AC/Au/C/I/A Temporal: E/RL/RC Ambiental: CR/IR/AR
EPSS Exploit Prediction Scoring System
Score predictivo Modelo probabilístico que predice la probabilidad de que una vulnerabilidad sea explotada en los próximos 30 días (de 0 % a 100 %). Complementar CVSS para priorizar remediación: una vulnerabilidad con CVSS alto pero EPSS bajo puede ser menos urgente que una con CVSS medio y EPSS muy alto. FIRST
Actualizado diariamente 		No reemplaza CVSS; lo complementa. Se aplica sobre CVEs existentes. Usado por el NVD y herramientas de VM modernas. EPSS: 0.94 (94 %)

▸ BASES DE DATOS Y REPOSITORIOS
NVD National Vulnerability Database
Base de datos Base de datos integral de vulnerabilidades sincronizada con la lista CVE, que añade metadatos enriquecidos: scores CVSS, disponibilidad de parches, referencias y más. Consultar vulnerabilidades con filtros avanzados (producto, versión, severidad) y obtener el score CVSS oficial de cada CVE. NIST
National Institute of Standards and Technology — EE. UU. 		CVE-ID ──▶ NVD (enriquece) NVD ──▶ CVSS score (base) NVD ──▶ CWE (clasificación) NVD ──▶ CPE (producto afectado) nvd.nist.gov
API pública disponible
KEV Known Exploited Vulnerabilities Catalog
Catálogo Catálogo de vulnerabilidades con explotación activa confirmada en el mundo real, mantenido por CISA. Identificar vulnerabilidades que ya están siendo usadas por actores de amenaza. Las agencias federales de EE. UU. están obligadas a parchearlas en plazos definidos. CISA
Cybersecurity and Infrastructure Security Agency 		Subconjunto de CVEs. Complementa CVSS/EPSS: una vuln en KEV debe priorizarse independientemente de su score. cisa.gov/kev
Actualización continua
▸ CLASIFICACIÓN Y TAXONOMÍA
CWE Common Weakness Enumeration
Taxonomía Lista categorizada de debilidades de software y hardware (fallas de diseño o implementación) que pueden originar vulnerabilidades. Clasificar el tipo de falla que causó una vulnerabilidad CVE para guiar la remediación y mejorar el diseño seguro. MITRE Corporation El NVD asocia un CWE a cada CVE. CWE describe la causa raíz; CVE describe la instancia concreta. No asigna severidad. CWE-79
Cross-site Scripting (XSS)
CWE-89: SQL Injection CWE-20: Improper Input
CPE Common Platform Enumeration
Esquema de nombres Esquema estandarizado de nombres para describir clases de aplicaciones, sistemas operativos y hardware de forma inequívoca. Identificar qué productos y versiones específicas están afectados por un CVE, facilitando la automatización del inventario y el análisis de impacto. NIST / NVD El NVD asocia CPEs a cada CVE. Herramientas de gestión de activos usan CPE para correlacionar inventario con vulnerabilidades. cpe:2.3:a:apache:log4j:2.14.1
ATT&CK Adversarial Tactics, Techniques & Common Knowledge
Framework Base de conocimiento de tácticas, técnicas y procedimientos (TTPs) usados por actores de amenaza reales, organizada por fases del ataque. Mapear cómo los atacantes explotan vulnerabilidades y qué controles de defensa aplicar en cada etapa de la cadena de ataque. MITRE Corporation Las CVEs pueden vincularse a técnicas ATT&CK específicas para contextualizar el impacto operacional de una vulnerabilidad.
T1190: Exploit Public App T1059: Command Execution
▸ PROGRAMAS Y PROCESOS
VDP Vulnerability Disclosure Program
Programa Política formal que define cómo investigadores externos pueden reportar vulnerabilidades a una organización de forma segura y responsable. Recibir reportes de vulnerabilidades de manera controlada antes de que sean publicadas como CVE o explotadas. Cada organización define el suyo (puede seguir estándares ISO 29147). Precede al proceso CVE: tras una divulgación responsable via VDP, el hallazgo puede recibir un CVE-ID de un CNA.
Bug Bounty Responsible Disclosure Coordinated Disclosure
VM Vulnerability Management
Proceso continuo Proceso cíclico de identificar, clasificar, priorizar, remediar y reportar vulnerabilidades en los sistemas de una organización. Reducir la superficie de ataque de forma sistemática y continua, usando CVE, CVSS, NVD y KEV como insumos de priorización. Interna de cada organización (apoyada en herramientas como Tenable, Qualys, Rapid7). Es el proceso que consume todos los demás términos: usa CVEs para identificar, CVSS/EPSS para priorizar, NVD/KEV para enriquecer, CWE para remediar la causa raíz.
